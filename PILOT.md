# SVX Pilot

## Objective

Demonstrate that a user can place an asset into trusted custody, track it, and retrieve it later.

---

## Pilot Participants

- 10 users
- 1 storage location
- 1 administrator

---

## User Flow

### Step 1 — Create Asset

User opens SVX.

User enters:

- Asset name
- Estimated value
- Photos
- Description

SVX creates an asset record.

---

### Step 2 — Select Custody

User selects a storage location.

Examples:

- Secure warehouse
- Verified location
- Trusted vault partner

SVX generates a custody request.

---

### Step 3 — Transfer Asset

User delivers the asset.

Storage location verifies receipt.

SVX records:

- Date
- Time
- Custodian
- Asset ID

Chain of custody begins.

---

### Step 4 — Asset Stored

Asset is secured.

User can view:

- Asset status
- Custody location
- Custody history

---

### Step 5 — Retrieval Request

User requests asset return.

SVX verifies ownership.

Retrieval is approved.

---

### Step 6 — Asset Returned

Asset is released.

Chain of custody is completed.

Pilot concludes successfully.

---

## Success Criteria

- Asset stored successfully
- Asset tracked successfully
- Asset retrieved successfully
- Chain of custody maintained
- User trusts the process
